# APP 测试方式指南

## 方法 1: Android Studio 模拟器（推荐）

### 安装步骤：
1. 下载安装 Android Studio: https://developer.android.com/studio
2. 打开 Android Studio
3. Tools → Device Manager → Create Device
4. 选择一个设备型号（推荐 Pixel 6）
5. 选择系统镜像（推荐 Android 13 或更高）
6. 点击 Finish 创建

### 使用方式：
```bash
# 启动模拟器后，在项目目录运行：
npx expo start
# 然后按 'a' 键自动在 Android 模拟器中打开
```

---

## 方法 2: iOS 模拟器（仅 Mac）

### 前置要求：
- macOS 系统
- 已安装 Xcode

### 使用方式：
```bash
# 启动 Expo 开发服务器
npx expo start

# 按 'i' 键自动在 iOS 模拟器中打开
```

---

## 方法 3: Windows 上的 Android 模拟器替代品

### A. 雷电模拟器（推荐国内用户）
- 下载：https://www.ldmnq.com/
- 轻量，启动快
- 需要在模拟器内安装 Expo Go APP

### B. 夜神模拟器
- 下载：https://www.yeshen.com/
- 界面友好，性能不错

### C. BlueStacks（蓝叠）
- 下载：https://www.bluestacks.com/
- 功能强大，但较占资源

### 使用步骤：
1. 安装并启动模拟器
2. 在模拟器的应用商店安装 "Expo Go"
3. 使用二维码或手动输入 URL

---

## 方法 4: 构建独立 APK（无需 Expo Go）

### 使用 EAS Build：
```bash
# 安装 EAS CLI
npm install -g eas-cli

# 登录 Expo 账号
eas login

# 配置项目
eas build:configure

# 构建开发版 APK
eas build --profile development --platform android

# 构建生产版 APK
eas build --profile production --platform android
```

下载生成的 APK 文件，直接安装到手机或模拟器。

---

## 方法 5: 直接构建本地 APK

### 使用 Expo 本地构建：
```bash
# 预构建原生代码
npx expo prebuild

# 使用 EAS 本地构建
eas build --platform android --local

# 或者使用 Android Studio
# 打开 android 目录，点击 Run
```

## 当前推荐方案

### 对于 Windows 用户：
1. **快速测试**：使用 Expo Go（手机扫码）
2. **深度测试**：安装 Android Studio 使用模拟器

## 常见问题

### Q: 模拟器连不上开发服务器？
A: 使用以下命令设置 ADB 反向代理：
```bash
adb reverse tcp:8081 tcp:8081
```

### Q: 想要测试推送通知等原生功能？
A: 必须构建独立 APK，Expo Go 有功能限制

### Q: 如何选择最适合的方式？
A: 
- 只是看效果 → Expo Go
- 需要调试原生功能 → Android Studio
- 快速演示 → 国产模拟器 + Expo Go
