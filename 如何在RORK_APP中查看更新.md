# 如何在 Rork App 中查看代码更新

## 🤔 为什么推送到 GitHub 后 App 没有更新？

### 重要概念：

```
GitHub 仓库 (代码存储)
    ↓
    需要构建/发布
    ↓
Rork App (运行的应用)
```

**推送代码到 GitHub ≠ 应用自动更新**

就像：
- 📝 修改菜谱（GitHub 代码）
- 🍳 但还没有做出新菜（Rork App）

---

## 🎯 两种查看更新的方法

### 方法 1：开发模式（推荐 - 即时查看）

**适用场景**：开发测试阶段，需要频繁查看修改

#### 步骤：

1. **启动开发服务器**（在您的电脑上）
   ```bash
   cd F:\picseek\rork--
   bun run start
   ```

2. **等待显示二维码或隧道 URL**
   - 终端会显示类似：`exp://192.168.x.x:8081`
   - 或显示一个二维码

3. **在 Rork App 中连接**
   
   **选项 A - 扫描二维码：**
   - 打开 Rork App
   - 点击"扫描二维码"
   - 扫描终端显示的二维码
   
   **选项 B - 手动输入地址：**
   - 打开 Rork App
   - 点击"手动输入开发服务器地址"
   - 输入显示的 URL（如 `exp://192.168.1.100:8081`）

4. **实时查看修改**
   - 每次保存代码后，App 会自动刷新
   - 看到您的所有修改（包括红色标题）

#### ⚠️ 注意事项：

- ✅ 手机和电脑需要在同一 WiFi 网络
- ✅ 开发服务器必须一直运行
- ✅ 修改代码后会自动热重载
- ❌ 关闭电脑或服务器，App 将断开连接

---

### 方法 2：生产模式（正式发布）

**适用场景**：准备发布给用户使用

#### 步骤：

1. **构建应用**
   ```bash
   cd F:\picseek\rork--
   npx eas-cli build --platform all
   ```

2. **提交到 Rork 平台**
   - 登录 Rork 开发者控制台
   - 上传构建好的应用
   - 提交审核

3. **发布更新**
   - 审核通过后
   - 用户打开 App 时会自动检查更新
   - 下载并安装新版本

#### ⏱️ 时间：
- 构建：10-30 分钟
- 审核：1-3 天
- 用户更新：自动或手动

---

## 🚀 当前建议：使用开发模式

**因为您正在开发和测试功能，应该：**

1. ✅ 保持开发服务器运行（`bun run start`）
2. ✅ 在 Rork App 中连接到开发服务器
3. ✅ 实时查看所有修改
4. ✅ 快速迭代和测试

**当功能完善后，再：**
1. 构建生产版本
2. 发布到 Rork App Store

## 📱 如何在 Rork App 中连接开发服务器

### 当前服务器状态：
✅ **开发服务器正在运行**
- 地址：`http://localhost:8081`
- 隧道已连接

### 连接步骤：

1. **在终端查看连接信息**
   - 查看终端输出
   - 会显示二维码或 URL

2. **打开 Rork App**
   - 点击"开发"或"Dev"选项
   - 选择"扫描二维码"或"输入 URL"

3. **扫描/输入**
   - 扫描二维码
   - 或输入显示的开发服务器地址

4. **等待加载**
   - App 会连接到您的开发服务器
   - 加载最新代码
   - 看到所有修改（包括红色标题）

## 💡 常见问题

### Q: 为什么我推送到 GitHub 但 App 没变化？
**A:** GitHub 只是代码仓库，App 需要重新构建或连接开发服务器。

### Q: 我必须每次都构建发布吗？
**A:** 不用！开发阶段使用开发模式，实时查看修改。只在准备发布时才构建。

### Q: 如何快速测试我的修改？
**A:** 
1. 运行 `bun run start`
2. 在 Rork App 中连接开发服务器
3. 修改代码自动刷新

### Q: 开发服务器断开后怎么办？
**A:** 重新运行 `bun run start`，然后在 Rork App 中重新连接。
